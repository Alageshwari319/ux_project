# UI-UX-EXPERIMENT5

## Aim:
To analyze two different UX project scenarios — a Smart Travel Booking Application and an Online Grocery Delivery Platform — by identifying user pain points, writing UX problem statements, selecting suitable UX research methods, and creating user personas in order to improve usability, accessibility, efficiency, and overall user satisfaction.

## Algorithm:

### Step 1: Select UX Project Scenarios

Choose two digital product scenarios for UX analysis:

Smart Travel Booking Application
Online Grocery Delivery Platform

### Step 2: Study the Existing User Experience

Analyze how users currently interact with the applications and identify common usability challenges.

### Step 3: Identify User Pain Points

Observe and list the major difficulties faced by users while using the applications such as:

Navigation problems
Checkout confusion
Delivery issues
Accessibility barriers
Information overload

### Step 4: Write UX Problem Statements

Create clear UX problem statements using the format:

“[User] needs a way to [achieve goal] because [obstacle], which results in [impact].”

This helps define the core usability problems affecting users.

### Step 5: Select Appropriate Research Methods

Choose suitable UX research techniques such as:

User Interviews
Surveys
Usability Testing
Contextual Inquiry
Diary Studies
Journey Mapping

Analyze why each method is appropriate for understanding user behavior and usability issues.

### Step 6: Conduct User Analysis

Study different types of users based on:

Age
Occupation
Technical skills
Usage behavior
User goals

### Step 7: Create User Personas

Develop detailed personas containing:

Demographic details
Goals
Behaviors
Pain points
Motivations

This helps represent real users and supports user-centered design decisions.

### Step 8: Evaluate Findings

Analyze all collected UX insights to identify:

Common usability issues
Accessibility concerns
User expectations
Opportunities for design improvement
